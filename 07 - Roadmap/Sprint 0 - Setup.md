# Sprint 0 — Setup (Semana 0)

## Objetivo
Tener los cimientos listos antes de escribir una línea de funcionalidad.

## Entregables

- Repositorios creados.
- Configuración base lista (TypeScript, base de datos, CI/CD).
- Un "Hello World" desplegado y accesible online.

## Definición de "terminado"

Cuando el equipo puede ver una página vacía servida desde producción, con backend que responde y base de datos conectada.

## Por qué este sprint existe

Resolver los problemas de infraestructura **antes** de meterse en producto evita que se conviertan en bloqueos durante los sprints de funcionalidades.
