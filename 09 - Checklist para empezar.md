# Checklist para empezar

Antes de escribir **la primera línea de código de funcionalidad**, confirma que tienes:

## Accesos

- [ ] Acceso al repo de GitHub.
- [ ] `.env` con todas las variables de entorno.
- [ ] Acceso al dashboard de streaming (con App ID).
- [ ] Acceso al dashboard de pagos (modo test).
- [ ] Acceso al proyecto de Firebase.
- [ ] Base de datos PostgreSQL funcionando.
- [ ] Redis funcionando.

## Diseño

- [ ] Diseños en Figma entregados por el diseñador.

## Conocimiento

- [ ] Especificación completa **leída entera** (ver [[00 - Índice|Índice]]).
- [ ] Filosofía y reglas de UX comprendidas.

## Legales

- [ ] NDA firmado.
- [ ] Contrato de cesión de propiedad intelectual firmado.

## Aprobación

- [ ] Approve del fundador (`yegaf1@gmail.com`) para empezar.

---

## Principios a recordar siempre

1. **Este documento es la fuente de verdad.** Si algo no está aquí, pregunta antes de inventar.
2. **Velocidad > perfección.** Es mejor entregar algo simple que funciona que algo complejo que no.
3. **Mobile-first siempre.** Diseña para celular primero.
4. **Pregunta cuando dudes.** El fundador prefiere responder 10 preguntas a tener que rehacer 1 funcionalidad.
5. **Tests no son opcionales** para los flujos críticos (pagos, streaming, moderación).
6. **El usuario no es el developer.** Diseña para personas que no saben nada de tecnología.
